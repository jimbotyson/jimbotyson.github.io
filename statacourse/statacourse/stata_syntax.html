Stata Syntax

Learning in this topic

The objective of this topic is to learn the basic, general syntax of Stata commands - what all commands will have in common.

Getting the format of basic commands under your belt will make using Stata more pleasurable and much faster.  In particular you should notice that a command has the general form:

command variable variable...variable, options

That is the variables passed to the command appear as a list separated by spaces and a comma precedes the list of options.

The essential activity in this topic is watching the video material and then working through the UCLA web page in Overview of Stata Syntax.

To try the example code in this topic, you will need to read the file results.xlsx from the website http://www.ucl.ac.uk/~ccaajim/results.xls

We call the structure of a Stata command the Stata syntax borrowing a term from grammar and computer programming.  The resources in this section will help you understand the common structure of Stata commands and show you examples of specific useful commands.

Some of the most common commands are those that operate on variables.  Their general form is

command varlist , options

The elements are varlist are variable names which are separated by spaces.  The options follow a single comma and are specified in the help for each command.

So

summarize maths english, detail

provides detailed summary statistics for the variables maths and english.

Often you can use the bysort command to group output by factors.  For example

bysort sex: summarize maths, detail

(There is also a by prefix command but to use it you must first sort by factor).

One command you need to start using right away is list.  List displays selections from the data in memory.  The bare list will show all variables for all cases.  But you can specify which variables to display in the varlist and you can use the if option to make selections of cases.

So you can

list surname maths, if maths>100

for example.

Try the following command

list in 1/5
